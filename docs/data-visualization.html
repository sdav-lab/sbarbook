9 Data Visualization with R

9.1 Introduction to Data Visualization

Data visualization is the graphical representation of information and data. In sport business analytics, effective visualizations transform complex datasets into clear, actionable insights that drive decision-making.

9.1.1 Why Visualize Data?

  1. Pattern Recognition: Humans process visual information faster than tables of numbers
  2. Communication: Visualizations make complex data accessible to stakeholders
  3. Decision Making: Clear visuals support faster, more confident decisions
  4. Storytelling: Charts and graphs tell compelling stories with data
  5. Exploration: Visualizations reveal patterns, trends, and outliers
# Load required packages
library(tidyverse)    # Includes ggplot2
library(ggplot2)      # Data visualization
library(Lahman)       # Baseball data
library(scales)       # Scale functions for ggplot2
library(gridExtra)    # Arrange multiple plots

9.1.2 Principles of Effective Visualization

Before creating visualizations, consider these fundamental principles:

1. Clarity: Make your message immediately obvious 2. Accuracy: Represent data truthfully without distortion 3. Efficiency: Maximize information with minimal ink 4. Aesthetics: Create visually appealing, professional graphics 5. Context: Provide appropriate labels, titles, and legends

9.1.3 Common Visualization Pitfalls to Avoid

# Example data
team_wins <- data.frame(
  team = c("Team A", "Team B", "Team C", "Team D"),
  wins = c(45, 50, 48, 52)
)

# BAD: Truncated y-axis (exaggerates differences)
bad_plot <- ggplot(team_wins, aes(x = team, y = wins)) +
  geom_bar(stat = "identity", fill = "steelblue") +
  coord_cartesian(ylim = c(40, 55)) +
  labs(title = "BAD: Truncated Y-axis",
       subtitle = "Makes small differences appear large") +
  theme_minimal()

# GOOD: Full y-axis (honest representation)
good_plot <- ggplot(team_wins, aes(x = team, y = wins)) +
  geom_bar(stat = "identity", fill = "steelblue") +
  labs(title = "GOOD: Full Y-axis",
       subtitle = "Honest representation of differences") +
  theme_minimal()

# Display side by side
gridExtra::grid.arrange(bad_plot, good_plot, ncol = 2)

9.2 Introduction to ggplot2

ggplot2 is R’s most powerful and popular data visualization package. It’s based on the “Grammar of Graphics,” which provides a systematic approach to creating visualizations.

9.2.1 The Grammar of Graphics

Every ggplot2 visualization is built from three essential components:

  1. Data: The dataset you want to visualize
  2. Aesthetics (aes): How variables map to visual properties (x, y, color, size, etc.)
  3. Geometries (geom): The type of plot (points, lines, bars, etc.)
# Basic ggplot2 structure
# ggplot(data = <DATA>, aes(x = <X_VAR>, y = <Y_VAR>)) +
#   geom_<TYPE>()

# Simple example
data(Teams)

# Filter recent MLB data
recent_teams <- Teams %>%
  filter(yearID == 2021)

# Create a simple scatter plot
ggplot(data = recent_teams, aes(x = R, y = W)) +
  geom_point() +
  labs(title = "Runs Scored vs. Wins (2021 MLB Season)",
       x = "Runs Scored",
       y = "Wins")

9.2.2 Building Plots Layer by Layer

ggplot2 uses a layered approach. You start with a base and add layers:

# Start with base layer
p <- ggplot(recent_teams, aes(x = R, y = W))

# Add points
p <- p + geom_point(size = 3, alpha = 0.6)

# Add regression line
p <- p + geom_smooth(method = "lm", se = TRUE, color = "red")

# Add labels
p <- p + labs(
  title = "Relationship Between Runs and Wins",
  subtitle = "2021 MLB Season",
  x = "Runs Scored",
  y = "Wins",
  caption = "Data: Lahman Baseball Database"
)

# Add theme
p <- p + theme_minimal()

# Display
print(p)

# Basic scatter plot
scatter1 <- ggplot(recent_teams, aes(x = RA, y = W)) +

gridExtra::grid.arrange(scatter1, scatter2, ncol = 2)

9.3.2 Bar Charts

Use case: Compare quantities across categories

# Prepare data: Top 10 teams by wins in 2021
top_teams <- recent_teams %>%
  arrange(desc(W)) %>%
  head(10) %>%
  mutate(team_name = paste(name, yearID))

# Basic bar chart
ggplot(top_teams, aes(x = reorder(name, W), y = W)) +
  geom_bar(stat = "identity", fill = "steelblue") +
  coord_flip() +  # Horizontal bars for readability
  labs(title = "Top 10 MLB Teams by Wins (2021)",
       x = "Team",
       y = "Wins") +
  theme_minimal()

# Grouped bar chart comparing wins and losses
team_records <- top_teams %>%
  select(name, W, L) %>%
  pivot_longer(cols = c(W, L), names_to = "outcome", values_to = "count")

ggplot(team_records, aes(x = reorder(name, -count), y = count, fill = outcome)) +
  geom_bar(stat = "identity", position = "dodge") +
  scale_fill_manual(values = c("W" = "darkgreen", "L" = "darkred"),
                    labels = c("Wins", "Losses")) +
  coord_flip() +

9.7.2 2. Choose the Right Chart Type

Data Type Visualization Use Case
Comparison Bar chart Compare values across categories

Relationship Scatter plot Examine correlation between variables

9.9 Chapter Summary

9.9.1 Key Concepts

  1. Principles of Visualization:
    • Clarity, accuracy, efficiency, aesthetics, context
    • Avoid truncated axes and misleading representations
    • Choose appropriate chart types for your data
  2. ggplot2 Grammar of Graphics:
    • Data + Aesthetics + Geometries = Visualization
    • Layer-by-layer construction
    • Consistent syntax across plot types
  3. Essential Plot Types:
    • Scatter plots: Relationships between continuous variables
    • Bar charts: Comparisons across categories
    • Line charts: Trends over time
    • Histograms: Distributions
    • Box plots: Comparing distributions, identifying outliers
  4. Customization:
    • Themes for consistent appearance
    • Color palettes for emphasis and branding
    • Annotations for highlighting insights
    • Faceting for small multiples
  5. Best Practices:
    • Know your audience (executives vs. analysts)
    • Maintain visual consistency
    • Tell stories with data
    • Export high-quality images for sharing
