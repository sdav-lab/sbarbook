Sport Business Analytics with R

10 Introduction to Tableau

10.1 Why Tableau for Sport Business Analytics?

While R provides powerful statistical analysis and visualization capabilities, Tableau excels at creating interactive dashboards and sharing insights with non-technical stakeholders. In sport business, you’ll often need to present data to executives, coaches, marketing teams, and sponsors who want interactive, visual reports rather than statistical code.

10.1.1 Tableau’s Role in Sport Analytics

Use Tableau when you need to: - Create interactive dashboards for stakeholders - Share live, updating reports with executives - Build visualizations quickly without coding - Present findings to non-technical audiences - Allow users to explore data themselves

Use R when you need to: - Perform complex statistical analyses - Run predictive models and machine learning - Automate repetitive analytical tasks - Work with very large datasets - Create reproducible research

Many sport analytics professionals use both tools: R for analysis, Tableau for presentation.

10.1.2 Real-World Sport Business Applications

  • Ticket Sales Dashboards: Track daily sales, pricing strategies, and revenue by section
  • Attendance Analytics: Visualize trends, compare against weather/opponent/promotions
  • Social Media Monitoring: Display engagement metrics, sentiment, reach
  • Player Performance: Interactive stats viewers for coaches and scouts
  • Sponsorship ROI: Show brand exposure, activation metrics, value delivered
  • Fan Demographics: Map season ticket holder locations, spending patterns

10.2 Getting Started with Tableau

10.2.1 Installing Tableau Public

For this course, we’ll use Tableau Public, a free version of Tableau Desktop perfect for learning and sharing visualizations publicly.

Download Tableau Public: 1. Visit https://public.tableau.com/ 2. Click “Download the App” 3. Create a free account 4. Install the application

Note: Tableau Public saves all work to the cloud publicly. For sensitive data, organizations use Tableau Desktop (paid) which allows local saving.

10.2.2 Tableau Interface Overview

When you open Tableau, you’ll see:

Area Purpose
Data Source Connect to data files (Excel, CSV, databases)
Worksheets Individual visualizations
Dashboards Combine multiple worksheets
Stories Create narrative sequences of dashboards
Dimensions Categorical fields (blue) - qualitative data
Measures Numeric fields (green) - quantitative data

10.2.3 Understanding Tableau Terminology

Term Definition Example
Worksheet A single visualization Bar chart of ticket sales by month
Dashboard Multiple worksheets combined Sales + attendance + revenue together
Story Sequence of dashboards with narrative Season performance story
Marks Visual elements representing data points Bars, lines, circles
Shelves Drop zones for fields (Rows, Columns, etc.) Drag “Sales” to Rows
Cards Control appearance (Color, Size, Label) Color by team, size by attendance
Filters Limit data shown Show only 2023 season

10.3 Connecting to Data

10.3.1 Loading a CSV File

Let’s create a sample dataset and load it into Tableau:

# Create sample sport business data
library(tidyverse)

# Sample ticket sales data
ticket_data <- data.frame(
  Date = seq.Date(from = as.Date("2023-01-01"),
                  to = as.Date("2023-12-31"),
                  by = "day"),
  Day_of_Week = weekdays(seq.Date(from = as.Date("2023-01-01"),
                                   to = as.Date("2023-12-31"),
                                   by = "day")),
  Month = month.name[month(seq.Date(from = as.Date("2023-01-01"),
                                     to = as.Date("2023-12-31"),
                                     by = "day"))],
  Tickets_Sold = sample(100:500, 365, replace = TRUE),
  Revenue = sample(100:500, 365, replace = TRUE) * runif(365, 35, 85),
  Category = sample(c("Season Tickets", "Single Game", "Group Sales"),
                    365, replace = TRUE),
  Section = sample(c("Lower Bowl", "Upper Bowl", "Club", "Suite"),
                   365, replace = TRUE)
)

# Save to CSV
write.csv(ticket_data, "sport_ticket_data.csv", row.names = FALSE)

In Tableau: 1. Open Tableau Public 2. Click “Connect” → “Text file” 3. Navigate to your CSV file 4. Tableau will preview your data 5. Click “Sheet 1” to start visualizing

10.3.2 Data Source Page

Before creating visualizations, review your data on the Data Source page:

  • Check field types: Tableau auto-detects data types (date, number, text)
  • Change if needed: Click the icon above field names to change types
  • Create calculations: Add new fields based on existing data
  • Join tables: Combine multiple data sources if needed

10.4 Building Your First Visualization

10.4.1 Creating a Bar Chart

Goal: Show total tickets sold by category

Steps: 1. Drag Category to Columns 2. Drag Tickets_Sold to Rows 3. Tableau creates a bar chart automatically 4. Click SUM(Tickets_Sold) → change to desired aggregation if needed

Customizing: - Color: Drag Category to Color in Marks card - Labels: Drag Tickets_Sold to Label - Sorting: Click sort icon in toolbar or right-click axis - Title: Double-click title to edit

10.4.2 Creating a Line Chart (Time Series)

Goal: Show daily ticket sales over time

Steps: 1. Drag Date to Columns 2. Drag Tickets_Sold to Rows 3. Tableau creates a line chart 4. Click the “+” next to YEAR(Date) to drill down to months, days

Important Concept: Blue vs. Green dates - Blue (Discrete): Treats each value separately (e.g., individual months) - Green (Continuous): Treats as a continuous timeline

To switch: - Right-click the date field → Choose between discrete and continuous options

10.4.3 Creating a Scatter Plot

Goal: Show relationship between tickets sold and revenue

Steps: 1. Drag Tickets_Sold to Columns 2. Drag Revenue to Rows 3. Each point represents a day 4. Drag Category to Color to see patterns by type 5. Drag Tickets_Sold to Size to vary point sizes

10.4.4 Working with Maps

Tableau has built-in geographic recognition for countries, states, cities, and even postal codes.

Goal: Map ticket sales by state (if you have geographic data)

Steps: 1. Drag State to the worksheet (Tableau creates a map automatically) 2. Drag Revenue to Color 3. Drag Tickets_Sold to Size 4. Tableau displays a filled map with color intensity showing revenue

Creating Hierarchies: For drill-down functionality (Country → State → City): 1. In Data pane, drag State onto Country 2. This creates a hierarchy 3. On map, click “+” to drill down from country to states 4. Click “+” again to drill to cities

10.5 Using Marks and Aesthetics

The Marks card controls how data appears visually.

10.5.1 Mark Type

Change visualization type by clicking the dropdown in Marks: - Automatic: Tableau chooses based on data - Bar: Bar charts - Line: Line charts - Area: Area charts - Pie: Pie charts - Circle: Scatter plots - Shape: Use shapes instead of circles - Text: Display values as text - Map: Geographic visualizations

10.5.2 Color

Color by Category: - Drag Category to Color - Tableau assigns colors automatically - Click Color → Edit Colors to customize palette

Color by Value: - Drag a measure like Revenue to Color - Tableau creates a gradient (low to high) - Click Color → Edit Colors to choose gradient colors

10.5.3 Size

Size by Value: - Drag Tickets_Sold to Size - Larger values = larger marks - Click Size slider to adjust overall size range

10.5.4 Labels

Show Values on Chart: - Drag field to Label - All marks will display values - Click Label → Font to customize appearance

10.5.5 Tooltip

Custom Hover Information: - Click Tooltip in Marks card - Edit text and add dynamic fields using Insert menu - Include relevant metrics for user context

Example tooltip:

Date: <Date>
Tickets Sold: <SUM(Tickets_Sold)>
Revenue: $<SUM(Revenue)>
Category: <Category>

10.6 Filters and Interactivity

10.6.1 Creating Filters

Basic Filter: 1. Drag Category to Filters shelf 2. Select values to include/exclude 3. Click OK

Show Filter to Users: 1. Right-click the field in Filters shelf 2. Select “Show Filter” 3. Filter appears on right side of worksheet 4. Users can now interact with it

Filter Types: - Categorical: Checkboxes for each value - Date: Calendar picker or relative dates - Numeric: Range slider or input boxes - Top N: Show only top/bottom N items

10.6.2 Date Filters

Relative Dates: 1. Drag Date to Filters 2. Select “Relative dates” 3. Choose range (Last 30 days, This year, etc.) 4. Data updates automatically as time passes

Date Range: 1. Drag Date to Filters 2. Select “Range of dates” 3. Choose start and end dates 4. Show filter for users to adjust

10.6.3 Quick Filters

Right-click any field and select “Show Filter” for instant filtering capability.

Filter Customization: - Right-click filter → Customize - Choose display type: dropdown, list, slider, etc. - Set default values - Allow multiple values or single select

10.7 Adding Analytics

10.7.1 Reference Lines

Add Average Line: 1. Switch to Analytics tab (next to Data tab) 2. Drag Average Line to visualization 3. Choose Table, Pane, or Cell level 4. Customize appearance and label

Add Trend Line: 1. From Analytics tab, drag Trend Line 2. Tableau fits a regression line to your data 3. Right-click trend line → Edit to see statistics 4. Choose linear, logarithmic, exponential, polynomial, or power models

10.7.2 Forecasting

Automatic Forecasting: 1. From Analytics tab, drag Forecast 2. Tableau uses time series methods to predict future values 3. Works best with consistent historical data 4. Right-click forecast → Describe Forecast for details

10.7.3 Calculations

Creating Calculated Fields: 1. Right-click in Data pane → Create Calculated Field 2. Write formula using Tableau functions 3. Example: [Revenue] / [Tickets_Sold] for average ticket price

Common Calculations for Sport Business:

// Average ticket price
[Revenue] / [Tickets_Sold]

// Percent of total
SUM([Tickets_Sold]) / TOTAL(SUM([Tickets_Sold]))

// Year-over-year growth
(SUM([Revenue]) - LOOKUP(SUM([Revenue]), -1)) / LOOKUP(SUM([Revenue]), -1)

// Attendance rate
[Actual_Attendance] / [Capacity]

// Win percentage
[Wins] / ([Wins] + [Losses])

10.8 Creating Dashboards

Dashboards combine multiple worksheets into a single interactive view.

10.8.1 Building a Dashboard

Steps: 1. Click Dashboard → New Dashboard (or icon at bottom) 2. Choose size (Desktop, Tablet, Phone, or custom) 3. Drag worksheets from left panel onto dashboard 4. Arrange and resize as needed

Layout Options: - Tiled: Worksheets snap to grid, resize automatically - Floating: Worksheets can overlap, positioned anywhere - Containers: Horizontal/Vertical containers organize related items

10.8.2 Dashboard Best Practices

1. Tell a Story: - Put most important metric at top-left (eye naturally goes there) - Arrange in logical flow - Use consistent color schemes across worksheets

2. Provide Context: - Include KPIs (Key Performance Indicators) prominently - Add titles and explanations - Show comparisons (vs. last year, vs. target, vs. league average)

3. Enable Exploration: - Use filter actions (clicking one chart filters others) - Add global filters for date ranges - Provide drill-down capabilities

4. Optimize Performance: - Limit number of worksheets (aim for 5-8 maximum) - Use extracts instead of live connections for large datasets - Hide unnecessary fields and calculations

10.8.3 Dashboard Actions

Filter Actions (most powerful feature): 1. Dashboard → Actions → Add Action → Filter 2. Choose source worksheet (user clicks here) 3. Choose target worksheets (these get filtered) 4. Run action on Select, Hover, or Menu

Example: Click a state on map → bar charts filter to show only that state’s data

Highlight Actions: - Emphasize related data points across sheets - Dashboard → Actions → Add Action → Highlight - Choose fields to highlight on

URL Actions: - Open websites or reports based on data - Useful for linking to player profiles, team pages, etc.

10.8.4 Adding Dashboard Objects

Text: - Drag “Text” from Objects - Add titles, descriptions, insights - Format with bold, colors, size

Images: - Drag “Image” from Objects - Add team logos, sponsor logos, photos - Set image size and positioning

Web Page: - Drag “Web Page” from Objects - Embed live web content - Example: Live Twitter feed, news ticker

Blank: - Drag “Blank” for spacing - Creates visual separation between elements

10.11 Tableau vs. R: When to Use Each

10.11.1 Strengths of R (ggplot2)

  • Complex statistical visualizations: Regression diagnostics, residual plots
  • Reproducibility: Code-based, version controlled
  • Customization: Infinite control over every element
  • Integration: Works seamlessly with statistical analyses
  • Publication: Create publication-quality static graphics
  • Automation: Generate hundreds of reports automatically

10.11.2 Strengths of Tableau

  • Interactivity: Built-in filters, drill-down, tooltips
  • Speed: Create visualizations in minutes without code
  • Sharing: Easy to share live dashboards with stakeholders
  • Exploration: Non-technical users can explore data themselves
  • Real-time: Connect to live databases for auto-updating dashboards
  • Presentation: Polished, professional appearance by default

10.11.3 Recommended Workflow

Typical Sport Analytics Project:

  1. Data Collection & Cleaning → R
  2. Statistical Analysis → R
  3. Initial Exploration → R (ggplot2)
  4. Dashboard Creation → Tableau
  5. Stakeholder Presentation → Tableau
  6. Ongoing Monitoring → Tableau (live dashboard)
  7. Deep Dives & Updates → Back to R as needed

Example: Season Ticket Sales Analysis - R: Calculate retention rates, identify at-risk accounts, build predictive model - Tableau: Create dashboard showing renewal status, demographics, segment performance - Result: Leadership monitors dashboard daily, data team updates model weekly in R

10.13 Tips for Success

10.13.1 Design Principles

1. Keep It Simple: - Don’t overload dashboards with too many charts - Focus on key metrics that drive decisions - Use white space effectively

2. Be Consistent: - Use same colors for same categories across charts - Maintain consistent fonts and sizes - Keep layout patterns familiar

3. Prioritize Information: - Most important metrics at top-left - Supporting details below or to the right - Use size to indicate importance

4. Guide the User: - Clear titles explaining what’s shown - Axis labels with units - Instructions for interactive elements - Annotations for key insights

10.13.2 Common Mistakes to Avoid

1. Misleading Axes: - Don’t truncate y-axis to exaggerate differences (unless clearly noted) - Start at zero for bar charts - Use consistent scales when comparing multiple charts

2. Too Much Interactivity: - Every filter/action should have a clear purpose - Don’t make users hunt for insights - Provide default views showing key findings

3. Poor Color Choices: - Avoid rainbow gradients (hard to interpret) - Use colorblind-friendly palettes - Don’t use color as only indicator (add patterns/shapes too)

4. Information Overload: - Maximum 7-8 worksheets per dashboard - Hide irrelevant fields - Aggregate when appropriate

5. Ignoring Context: - Always provide comparison (vs. last year, vs. target) - Show sample sizes (100% of what?) - Include data source and date updated

10.14 Advanced Features (Optional)

10.14.1 Parameters

What: User-controlled values that can modify calculations, filters, reference lines

Example: Create a “Target Revenue” parameter - Users adjust target value - Dashboard shows actual vs. target - Colors change based on meeting target

10.14.2 Sets

What: Custom groups of dimension members

Example: “Top 10 Customers” set - Automatically updates as data changes - Use in filters or calculated fields - Create in/out groups dynamically

10.14.3 Table Calculations

What: Calculations performed on query results, not source data

Examples: - Running total - Percent of total - Year-over-year growth - Moving average - Rank

Use: Show trends, growth rates, rankings without modifying source data

10.14.4 LOD Expressions

What: Level of Detail expressions for calculations at different granularities

Example:

{FIXED [Customer] : SUM([Revenue])}

Calculates total revenue per customer, regardless of view granularity
