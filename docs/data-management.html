5 Data Management and Preparation in R

Key Terms Quick Reference

Essential data management terminology for this chapter:

Term Definition Example
Working Directory The folder where R looks for files by default /Users/sean/Documents/Sport_Analytics/
Package Collection of functions that extend R’s capabilities dplyr package adds data manipulation functions
Library Command to load an installed package library(dplyr)
Data Frame R’s version of a spreadsheet (rows and columns) Player stats with columns for name, points, assists
Vector Collection of values of the same type c(25, 30, 22, 28) - points scored
Factor Categorical variable with defined levels Conference names: “SEC”, “ACC”, “Big Ten”
NA Missing value in R Player stats where assists = NA (not recorded)
Pipe Operator (%>%) Chains functions together, reads as “then” data %>% filter() %>% select()
Tidy Data Each variable is a column, each observation is a row One row per player, columns for stats
CSV Comma-Separated Values file format ticket_sales.csv

Why This Matters to Sport Business: Messy data leads to wrong conclusions. In sport analytics, you’ll spend 60-80% of your time cleaning and organizing data before analyzing it. Mastering data management saves hours and prevents costly mistakes like analyzing incomplete data or using wrong column values.

5.1 Understanding Your R Environment

Before we dive into data analysis, it’s essential to understand how to manage your R environment effectively. This includes setting up your workspace, understanding file paths, and organizing your analytical projects.

5.1.1 The RStudio Interface

RStudio provides four main panes that help you work efficiently:

  1. Source Editor (top-left): Where you write and edit R scripts
  2. Console (bottom-left): Where R executes commands and displays output
  3. Environment/History (top-right): Shows loaded datasets and command history
  4. Files/Plots/Packages/Help (bottom-right): File browser, plot viewer, and help documentation
# You can run code from the Source Editor by:
# 1. Placing cursor on the line and pressing Ctrl+Enter (Windows) or Cmd+Enter (Mac)
# 2. Selecting multiple lines and pressing Ctrl+Enter or Cmd+Enter
# 3. Clicking the "Run" button in RStudio

5.1.2 Working Directory Management

Your working directory is the folder where R looks for files and saves output by default.

# Check your current working directory
getwd()
## [1] "/home/sean/Documents/claude/Course Books/SPM 4703"
# Set a new working directory (modify path as needed)
# setwd("~/Documents/Sport_Analytics")  # Mac/Linux
# setwd("C:/Users/YourName/Documents/Sport_Analytics")  # Windows

Best Practice: Create a dedicated folder for each R project and set it as your working directory at the start of each session.

5.2 Installing and Loading Packages

R’s functionality extends through packages - collections of functions, data, and documentation created by the R community.

5.2.1 Installing Packages

You only need to install a package once on your computer:

# Install the tidyverse suite of packages
install.packages("tidyverse")

# Install ggplot2 for data visualization
install.packages("ggplot2")

# Install multiple packages at once
install.packages(c("dplyr", "readr", "tidyr"))

5.2.2 Loading Packages

After installation, you must load packages in each R session before using them:

# Load tidyverse (includes dplyr, ggplot2, readr, and more)
library(tidyverse)

# Load ggplot2 specifically
library(ggplot2)

Important: You install once but load every session.

5.3 Loading and Exploring Data

5.3.1 Reading CSV Files

CSV (Comma-Separated Values) files are the most common format for sport analytics data.

# Basic method - assumes file is in working directory
data <- read.csv("Chapter 2 data.csv")

# Using file.choose() - opens file browser dialog
data <- read.csv(file.choose())

# Using full file path
data <- read.csv("~/Documents/Sport_Analytics/Chapter 2 data.csv")

5.3.2 Exploring Dataset Structure

Once data is loaded, explore its structure using these essential functions:

# Display first 6 rows
head(data)

# Display last 6 rows
tail(data)

# Show data structure and types
str(data)

# Get summary statistics for all variables
summary(data)

# Display column names
colnames(data)

# Check number of rows
nrow(data)

# Check number of columns
ncol(data)

# Get dimensions (rows, columns)
dim(data)

Understanding Factors:

Factors are R’s way of storing categorical data (like conference names, player positions, or team names). Think of factors as text with special properties:

  • R stores factors as numbers internally (1 = “ACC”, 2 = “Big Ten”, 3 = “SEC”)
  • Displays them as text labels for readability
  • Remembers all possible categories (called “levels”), even if some don’t appear in your data
  • Efficient for large datasets (stores “SEC” 1,000 times as number 3 repeated, not the text)
  • Useful for statistical analyses that treat categories differently than regular text

When you see stringsAsFactors = FALSE: This prevents R from automatically converting text to factors. In older R versions, text was automatically converted to factors, which could cause unexpected behavior. Now you have control:

# With stringsAsFactors = FALSE (recommended)
data <- data.frame(team = c("Lakers", "Warriors"),
                   wins = c(50, 55),
                   stringsAsFactors = FALSE)  # team stays as text

# team column is character (text) - easier to manipulate

calculate_player_efficiency <- function(points, rebounds, assists,

# Create a function library for common calculations
sport_business_features <- function(df) {
  df %>%
    mutate(
      # Revenue metrics
      revenue_per_fan = if ("total_revenue" %in% names(.) & "attendance" %in% names(.))
        round(total_revenue / attendance, 2) else NA,

      # Capacity utilization
      capacity_pct = if ("attendance" %in% names(.) & "capacity" %in% names(.))
        round((attendance / capacity) * 100, 1) else NA,

      # Time-based features
      is_weekend = if ("game_date" %in% names(.))
        wday(game_date) %in% c(1, 7) else NA
    )
}
