3 Sport Data Sources

3.1 Public Sport Databases

The foundation of sport analytics lies in accessing quality data. Fortunately, numerous public databases exist that provide rich datasets for analysis.

3.1.1 Lahman Baseball Database

The Lahman Baseball Database is one of the most comprehensive collections of baseball data available. It contains:

  • Player statistics from 1871 to 2022
  • Team rosters and records
  • Manager and umpire data
  • Salaries and contracts
# Install the Lahman package
install.packages("Lahman")

# Load the package
library(Lahman)
# Load sample data

3.3 API Integration for Real-time Data

Application Programming Interfaces (APIs) provide direct access to sports data from various providers.

3.3.1 What is an API?

An API is a way for computer programs to talk to each other. Instead of visiting a website and clicking buttons to get data, an API lets your R code request data directly from a server.

Analogy: Think of an API like a restaurant menu: - The menu (API documentation) tells you what data you can request - You place an order (API call) specifying what you want - The kitchen (server) prepares it - The waiter (API) brings you the data

Example: Instead of manually copying player stats from ESPN.com, your R code can use ESPN’s API to automatically fetch the latest stats every hour.

Benefits for Sport Analytics: - Automated: Get data without manual copying - Current: Access real-time or frequently updated data - Structured: Data arrives in clean, consistent formats (usually JSON or XML) - Scalable: Request data for 1 player or 1,000 players with same code

3.3.2 Sportradar API

Sportradar provides APIs for real-time sports data including:

  • Live scores and statistics
  • Player and team information
  • Historical data sets

3.4 Data Quality and Cleaning Considerations

When working with sport data, several quality considerations must be addressed:

3.4.1 Missing Data Issues

# Create sample data with missing values
sample_player_data <- data.frame(
  player = c("Player A", "Player B", "Player C", "Player D"),
  points = c(22, 18, NA, 25),
  assists = c(5, NA, 8, 6),
  rebounds = c(4, 6, 3, NA),
  stringsAsFactors = FALSE
)

# View missing values
is.na(sample_player_data)
##      player points assists rebounds
## [1,]  FALSE  FALSE   FALSE    FALSE
## [2,]  FALSE  FALSE    TRUE    FALSE
## [3,]  FALSE   TRUE   FALSE    FALSE
## [4,]  FALSE  FALSE   FALSE     TRUE
# Remove rows with missing data
complete_cases <- sample_player_data[complete.cases(sample_player_data), ]
complete_cases
##     player points assists rebounds
## 1 Player A     22       5        4

# Function to generate daily sales report
generate_daily_sales_report <- function(date) {
  # Load data (from database, API, or file)
  sales <- read.csv(paste0("sales_", date, ".csv"))

  # Calculate key metrics
  report <- sales %>%
    summarize(
      total_transactions = n(),
      total_revenue = sum(total_price),
      avg_transaction = mean(total_price),
      unique_customers = n_distinct(customer_id)
    )

  # Export report
  write.csv(report, paste0("daily_report_", date, ".csv"), row.names = FALSE)

  return(report)
}

# Example: Anonymizing customer data
customer_data_raw <- data.frame(
  customer_id = paste0("C", 1001:1005),
  name = c("John Smith", "Jane Doe", "Bob Johnson", "Alice Williams", "Charlie Brown"),
  email = c("john@email.com", "jane@email.com", "bob@email.com",
            "alice@email.com", "charlie@email.com"),
  total_spend = c(5000, 12000, 3500, 8500, 6200)
)

# Anonymize for analysis
customer_data_anonymized <- customer_data_raw %>%
  mutate(
    customer_id = paste0("ANON_", row_number()),  # Replace real ID
    name = NULL,  # Remove PII
    email = NULL  # Remove PII
